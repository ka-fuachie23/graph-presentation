Graph data structure

By Kwabena Afari Fuachie

What is a graph

  • It is a collection of nodes and edges
  • Nodes contain data
  • Edges are connections between nodes

Types of graphs

  • Directed and undirected graphs
  • Weighted and unweighted graphs

Terminologies

  • Vertices
  • Adjacent nodes
    • Adjacent nodes are neighbours
  • Cyclic and acyclic paths

Use cases

  • Describing relationships between things
  • Representing State transition diagrams

Representation

  • Adjacency matrix
  • Adjacency list

Adjacency matrix

  • A 2d matrix of nodes with connections represented by a boolean value
  • Most helpful for small graphs or almost complete graphs
  • Space complexity: O(V2)
  • Time complexity - Connection checking: O(1)
  • Time complexity - Neighbours finding: O(V)

Adjacency list

  • An array of the lists representing connections to each node
  • Most useful if continuous access of a node's connections is needed
  • Space complexity: O(V + E)
  • Time complexity - Connection checking: O(Nu)
  • Time complexity - Neighbours finding: O(Nu)

Traversal Algorithms

  • Depth First Traversal
  • Breadth First Traversal

Depth First Traversal

  • Explores nodes as far as possible down a path
  • Does not always guarantee the shortest path

Breadth First Traversal

  • Explores nodes equally in all directions
  • Always guarantees the shortest path
